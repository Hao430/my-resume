## 🚀 CI/CD 工作流说明

### 工作流文件: `.github/workflows/ci-cd.yml`

#### 触发条件
| 事件 | 分支/标签 | 说明 |
|------|----------|------|
| `push` | `main`, `develop` | 推送代码时触发 |
| `push` | `v*` 标签 | 推送版本标签时触发 |
| `pull_request` | `main`, `develop` | PR 创建/更新时触发 |

#### Job 流程图

```
┌─────────────────┐
│  1. Quality     │  Lint + TypeScript 类型检查
│     Check       │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  2. Build Test  │  生产构建 + 产物上传
│                 │
└────────┬────────┘
         │
    ┌────┴────┬────────────┐
    ▼         ▼            ▼
┌────────┐ ┌────────┐  ┌────────┐
│ Deploy │ │Release │  │ Notify │
│(main)  │ │(tag)   │  │(main)  │
└────────┘ └────────┘  └────────┘
```

#### 各 Job 详细说明

##### 1. Quality Check (代码质量检查)
- **运行环境**: `ubuntu-latest`
- **步骤**:
  1. 检出代码
  2. 安装 pnpm
  3. 设置 Node.js 22
  4. 安装依赖 (`pnpm install --frozen-lockfile`)
  5. Lint 检查 (`pnpm run lint`)
  6. TypeScript 类型检查 (`pnpm run type-check`)

##### 2. Build Test (构建测试)
- **依赖**: 需 Quality Check 通过
- **步骤**:
  1. 生产构建 (`pnpm run build-only`)
  2. 上传构建产物到 Artifacts (保留 7 天)

##### 3. Deploy (部署到生产环境)
- **触发条件**: 仅 `main` 分支推送
- **依赖**: Quality Check + Build Test 通过
- **环境**: production (带审批保护)
- **说明**: 
  - 当前项目使用阿里云 ESA 自动部署
  - GitHub Actions 执行构建验证
  - ESA 监听 GitHub 仓库变化自动部署

##### 4. Release (创建 GitHub Release)
- **触发条件**: 推送 `v*` 标签 (如 `v1.0.0`)
- **依赖**: Quality Check + Build Test 通过
- **输出**: 
  - 自动生成 Release Notes
  - 附带构建产物压缩包

##### 5. Notify (部署通知)
- **触发条件**: Deploy Job 完成后
- **输出**: 控制台打印部署状态信息

## 🌐 部署架构

### 当前部署方式: 阿里云 ESA

```
GitHub Repo (main)
       │
       │ 推送代码
       ▼
┌──────────────┐
│ 阿里云 ESA   │ 自动监听 GitHub 变更
│  Pages      │
└──────┬───────┘
       │
       │ 自动构建 & 部署
       ▼
┌──────────────┐
│  hao430.cn   │ 生产环境
└──────────────┘
```

### ESA 配置 (`esa.jsonc`)
```json
{
  "name": "hao430",
  "installCommand": "npm install -g pnpm && pnpm install --frozen-lockfile",
  "buildCommand": "pnpm run build",
  "assets": {
    "directory": "./dist",
    "notFoundStrategy": "singlePageApplication"  // 支持 Vue Router History 模式
  }
}
```

## 🔄 开发工作流建议

### 日常开发
1. 创建功能分支: `git checkout -b feature/xxx`
2. 开发并提交: `git commit -m "feat: xxx"`
3. 推送分支: `git push origin feature/xxx`
4. 创建 PR 到 `develop` 分支
5. CI 自动运行检查

### 发布流程
1. 合并 PR 到 `main` 分支
2. CI 自动运行并部署到生产环境
3. 打标签发布: `git tag v1.0.0 && git push origin v1.0.0`
4. GitHub Actions 自动创建 Release

## ⚠️ 注意事项

1. **Node 版本**: 项目要求 Node ^20.19.0 或 >=22.12.0
2. **包管理器**: 强制使用 pnpm，CI 中已配置
3. **依赖锁定**: 使用 `pnpm-lock.yaml`，CI 使用 `--frozen-lockfile`
4. **SPA 配置**: ESA 已配置 `singlePageApplication` 策略，支持前端路由刷新
5. **Git 推送**: 日常开发小更新只提交到本地，明确说"推送"时才执行 `git push`
