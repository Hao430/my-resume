## 三、架构分析

### 3.1 项目结构 ✅ 优秀

```
src/
├── assets/styles/     # 设计系统（variables.css + global.css）
├── components/        # 9个可复用组件
├── data/              # 简历数据（静态 JSON）
├── pages/             # 6个页面组件（新增 SlideViewerPage）
├── router/            # 路由配置
├── stores/            # Pinia 状态管理（blog.ts + slides.ts）
└── types/             # TypeScript 类型定义
```

**优点**：
- 组件与页面分离，职责清晰
- 设计系统独立，易于主题切换
- 类型定义完整
- Store 架构统一管理博客和演示数据

### 3.2 路由设计 ✅ 完善

```typescript
/                  → HomePage.vue（首页：项目预览 + 统计）
/about             → AboutPage.vue（完整简历）
/blog              → BlogPage.vue（博客列表）
/blog/:slug        → BlogPostPage.vue（博客详情）
/slides            → SlidesPage.vue（演示文稿列表）
/slides/:id        → SlideViewerPage.vue（演示文稿查看器） ✅ 新增
```

**优点**：
- 懒加载配置（代码分割）
- 页面过渡动画
- 演示文稿查看器使用 iframe 嵌入独立 HTML

### 3.3 状态管理 ✅ 已完善

**现状**：
- `blog.ts` - 博客数据 Store，提供 getPostBySlug、addPost 方法
- `slides.ts` - 演示数据 Store，提供 getSlideById、addSlide 方法
- `data/resume.ts` - 简历数据保持静态

**改进点**：
- 博客/演示数据集中管理
- 统一数据更新入口
- TypeScript 类型完整

### 3.4 数据层 ✅ 良好

**改进后**：
- 博客数据迁移至 `stores/blog.ts`
- 演示数据迁移至 `stores/slides.ts`
- 简历数据保留在 `data/resume.ts`

**未来可扩展**：
- 考虑引入 Markdown 文件支持（Vite Plugin MD）
- 或接入轻量级 CMS

### 3.5 设计系统 ✅ 优秀

**变量体系**：
```css
--color-*      /* 12个颜色变量 */
--space-*      /* 12个间距级别 */
--text-*       /* 8个字号级别 */
--radius-*     /* 4个圆角级别 */
--transition-* /* 4个过渡时长 */
```

**优点**：
- 墨砚风格（朱砂红 + 象牙白）主题独特
- CSS 变量覆盖全面
- 动画系统完善（6种动画 + 延迟类）
